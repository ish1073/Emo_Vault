## 🗄️ Database Schema

### Table: `regrets`
```sql
- regret_id (Primary Key)
- user_id (Foreign Key)
- description (VARCHAR 500)
- lesson_learned (VARCHAR 500)
- tag (VARCHAR 50) - Category (procrastination, stress, communication, etc.)
- created_date (TIMESTAMP)
```

**Purpose:** Store user regrets with lessons learned and categorization for pattern analysis.

---

### Table: `habits`
```sql
- habit_id (Primary Key)
- user_id (Foreign Key)
- name (VARCHAR 200)
- description (VARCHAR 500)
- suggested_by_tag (VARCHAR 50) - Link to regret category
- is_active (BOOLEAN)
- created_date (TIMESTAMP)
```

**Purpose:** Store habits that users are building. Habits can be auto-suggested based on regret patterns.

---

### Table: `habit_logs`
```sql
- log_id (Primary Key)
- habit_id (Foreign Key)
- completed_date (DATE)
- is_completed (BOOLEAN)
```

**Purpose:** Track daily habit completion to calculate streaks and consistency scores.

---

### Table: `alerts`
```sql
- alert_id (Primary Key)
- user_id (Foreign Key)
- alert_type (VARCHAR 50) - Type: REGRET_PATTERN, HIGH_STRESS, HABIT_CONSISTENCY, HABIT_STREAK_BROKEN
- message (VARCHAR 500)
- is_read (BOOLEAN)
- created_date (TIMESTAMP)
```

**Purpose:** Store behavioral risk alerts generated by RiskAnalyzer.

## 🧠 Java Architecture

### Model Layer (3 Classes)

#### 1. **Regret.java**
- Fields: regretId, userId, description, lessonLearned, tag, createdDate
- Represents a single regret entry with associated lesson

#### 2. **Habit.java**
- Fields: habitId, userId, name, description, suggestedByTag, isActive, currentStreak, consistencyScore
- Stores habit data with calculated metrics

#### 3. **Alert.java**
- Fields: alertId, userId, alertType, message, isRead, createdDate
- Represents behavioral risk alerts

---

### Data Access Layer (3 Classes)

#### 1. **RegretDAO.java**
```
CRUD Methods:
- addRegret(Regret)
- getAllRegretsByUserId(int userId)
- getRegretById(int regretId)
- updateRegret(Regret)
- deleteRegret(int regretId)

Analysis Methods:
- getTagFrequency(int userId) - Returns map of tags and counts
- getMostCommonTag(int userId) - Returns most frequent tag
```

#### 2. **HabitDAO.java**
```
CRUD Methods:
- addHabit(Habit)
- getAllHabitsByUserId(int userId)
- getHabitById(int habitId)
- updateHabit(Habit)
- deleteHabit(int habitId)

Tracking Methods:
- completeHabitToday(int habitId)
- isCompletedToday(int habitId)

Calculation Methods:
- calculateStreak(int habitId) - Consecutive completed days
- calculateConsistencyScore(int habitId) - % of days done in last 30 days
```

#### 3. **AlertDAO.java**
```
Creation & Retrieval:
- addAlert(Alert)
- getUnreadAlertsByUserId(int userId)
- getAllAlertsByUserId(int userId)
- getUnreadAlertCount(int userId)

Management:
- markAsRead(int alertId)
- deleteAlert(int alertId)
```

---

### Business Logic Layer

#### **RiskAnalyzer.java** - Utility Class
```
Analysis Methods:
1. analyzeRegretRisks(int userId)
   - Checks for repeated regret tags (≥3 occurrences)
   - Generates alert: "Risk of repeating X behavior"

2. analyzeStressRisks(int userId)
   - Checks emotion entries for stress markers
   - If ≥5 high-stress emotions in 7 days: generates alert

3. analyzeHabitRisks(int userId)
   - Checks consistency score (<50%)
   - Checks for broken streaks
   - Generates appropriate alerts

4. generateAllRiskAlerts(int userId)
   - Combines all three analyses
   - Returns list of all active alerts

Suggestion Method:
5. suggestHabitsFromRegrets(int userId)
   - Analyzes most common regret tags
   - Returns suggested habits for each tag
   ```

Example Suggestions:
- Procrastination → "Start tasks early", "Time blocking"
- Stress → "Daily breathing", "Daily walk"
- Communication → "Speak up in meetings", "Weekly check-in"
```

### Controller Layer (3 Servlets)

#### 1. **RegretServlet** (URL: `/regret`)
```
GET:  Fetch all regrets, tag frequency, and insights for user
POST: Add new regret OR Delete existing regret

Actions:
- action=add (Description, Tag, Lesson)
- action=delete (Regret ID)
```

#### 2. **HabitServlet** (URL: `/habit`)
```
GET:  Fetch all habits and habit suggestions for user
POST: Add new habit, Complete habit today, OR Delete habit

Actions:
- action=add (Name, Description, Tag)
- action=complete (Habit ID) - Mark as done today
- action=delete (Habit ID)
```

#### 3. **AlertServlet** (URL: `/alert`)
```
GET:  Fetch alerts (unread or all) and generate fresh risk alerts
POST: Mark alert as read OR Delete alert

Actions:
- action=read (Alert ID)
- action=delete (Alert ID)

Parameters:
- view=unread (default) or view=all
```

## ⚡ Performance Considerations

### Optimizations Implemented:
1. **Database Indexes:**
   - User_id indexed on all tables
   - Tag indexed on regrets
   - completed_date indexed on habit_logs

2. **Calculation Efficiency:**
   - Streak calculation optimized
   - Consistency cached in Habit object
   - Alert generation batched

3. **Query Optimization:**
   - Prepared statements prevent SQL injection
   - Limited alert retrieval (TOP 5)
   - Joins minimal

## 🔒 Security

### Implemented:
1. **Session Validation:**

## 🚨 Troubleshooting

### Issue: 404 error on /regret page
**Solution:** Check web.xml servlet mappings are added and Tomcat is restarted

### Issue: Database connection error
**Solution:** Run SQL schema setup on emovault database

### Issue: No alerts showing
**Solution:** Add regrets or habits first; alerts are generated on-demand

### Issue: Streak not updating correctly
**Solution:** Check system date; streak is based on consecutive calendar days
